## 3. Decisions Made
- The dev build should launch the unified prototype by default.
  - Why: the migration needed a visible default path for validation and iteration.
  - Rejected alternative: keep the legacy runtime path as the dev default and relegate the unified prototype to opt-in only.
- Prototype modules/binaries should stay gated behind the `ux-prototypes` feature.
  - Why: this keeps the default runtime build surface smaller and avoids accidental coupling.
  - Rejected alternative: make the prototype paths always-on.
- Shared hint copy and key tokens should be centralized in `src/ui/input_hints.rs`.
  - Why: the runtime panels were drifting and needed one canonical source of truth.
  - Rejected alternative: leave duplicate literal hint strings in each panel.
- The failing rollback contract should assert the real preservation invariant across inventory plus equipped slot.
  - Why: the inventory-only count was wrong for a case where one copy remains equipped after rollback.
  - Rejected alternative: keep the inventory-only assertion and treat the test as a rollback failure.
- Runtime UI docs should continue to reflect the unified prototype as the preferred target.
  - Why: the migration should not fork the design language in the documentation layer.
  - Rejected alternative: let the runtime docs and prototype diverge.
